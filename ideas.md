1. Использовать косинусную схожесть для определения соответствия комментария теме поста.
2. Для каждого пользователя составлять "рейтинг" на основе его комментариев, жалоб на них, его жалоб на другие комментарии и тд.
3. Использовать бизнес метрики вида:
    - User Retention Rate — процент пользователей, которые продолжают активно участвовать в обсуждениях.
    - Engagement Quality — метрики качественной активности (длина комментариев, конструктивные обсуждения, upvotes на позитивный контент).
    - Moderation Accuracy Rate — процент корректно модерированного контента.
    - Response Time to Flagged Content — среднее время реакции на жалобы.
4. Использовать модель не только (вообще не) во время написания комментария, а, например, при жалобе от пользователя.
5. Использовать совместно со словарем profanity-filter, для учета f@ck и тд случаев.
6. Возможно построить еще одну модель для опеделения соответствия комментария теме поста.
